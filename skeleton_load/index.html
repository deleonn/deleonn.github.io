Thoughts about skeleton screen by @deleonn

What is Skeleton Load?

Skeleton load is the name I used to call a modern way to indicate loading processes in any website, althought the web community just called it skeleton screen. It is a modern and pleasing-to-the-eye alternative for the old and boring loading spinners.

According to the definitions given through many blogs and websites, the skeleton load or skeleton screen it's a version of a page to be rendered by the browser for the users to see it, in which the main content or information is going to be loaded.

The user @basham commented on an issue opened in GitHub stating that with a skeleton screen, the focus is on content being loaded not the fact that its loading and thatâ€™s real progress. You can watch the entire post here.

Quick fact: at the time, you might already now this but Facebook, Nintendo, Apple and many others are using skeleton screens.

Why is it important?

The skeleton load it's a way to convert the loading experience we already know into information, allowing the user to be prepared to the content that is going to be rendered. It's like saying: Hey there, it doesn't matter if this is your first time here, we are going to display information in this skeleton so you can easily distinguish where to look at.

Watch a live example I made myself, using pure css and AngularJS to simulate an ajax call with the $timeout function. The skeleton is the first thing rendered, and after 2 seconds, it is removed from the DOM and the simulated fully loaded ajax response is displayed.

Conclusion

User Experience is something we, as developers, need to take care of and give the importance it deserves. I've experienced myself many divided arguments between web developers about how important backend all alone is, or that frontend developers or web designers shouldn't be treated as respectable developers. Whatever your opinion is, skeleton load is an example of how web design -and I'm not talking about pure design- actually matters to every dev out there, to make our projects or applications more enjoyable to the users, as that may be the most important thing of web development. Things like this should be included at the Web Developing 101: Best Practices books all over the place. It would give the new guys the ability to create very nice and simple user interfaces without having too much knowledge in their firsts projects. If loading spinners are such easy things, why not the skeleton screens?
